# Break Reminder (Android)

A tiny, lightweight Flutter app that reminds you to take a break — on your schedule. Each reminder fires as a notification with **Start break** / **Skip** actions:

- **Start break** opens the app straight into a stopwatch. When you're done, tap **End break** — the break (start, end, duration) is logged to a local SQLite database and the next reminder is scheduled one interval (default 1 hour) after the break ended.
- **Skip** logs a skipped break without opening the app; the reminder cadence continues unchanged.

Reminders only fire inside your **active schedule**: pick which weekdays it runs (e.g. only Monday and Wednesday) and a daily time window (e.g. 9:00 AM – 6:00 PM). A reminder that would land outside the window rolls forward to the next active day's window start.

The home screen is a control panel: stat tiles (breaks today, break time today/this week/all-time), a week/month/year trend chart of break minutes, and a GitHub-style year heatmap with streak / best-day / year-total callouts.

## How scheduling works (design note)
Because the cycle re-anchors when a break ends, a single repeating notification can't model it. The app keeps a **chain of one-shot scheduled notifications** (the next ~24 occurrences, computed inside the active window):
- Ignoring or skipping a reminder costs nothing — the rest of the chain still fires.
- Ending a break cancels the chain and reschedules it anchored at break end.
- The chain is topped back up on app launch when it runs low, so it can't run dry in normal use. Scheduled notifications survive reboots (handled by `flutter_local_notifications`' boot receiver).

## Project layout
- `lib/core/db/` — SQLite layer (`sqlite3` via FFI, safe in the background notification isolate). Table: `breaks(id, start_ts, end_ts, duration_seconds, status)` with status `completed` or `skipped`.
- `lib/core/schedule/` — pure schedule math (active days + daily window + interval → next reminder instants).
- `lib/core/notifications/` — scheduling chain + Start break / Skip handling, including while the app isn't running.
- `lib/core/settings/` — pause/resume, interval, active days, active hours, in-progress break persistence.
- `lib/features/home/` — control panel (stat tiles, trend chart, heatmap, start-break button).
- `lib/features/break/` — the stopwatch screen.
- `lib/features/settings/` — pause toggle, interval stepper, weekday chips, time-window pickers, test reminder.

The Android-native scaffold (`android/`) isn't committed — it should be generated by the Flutter tool itself so it matches your installed Flutter version (step 2 below).

## Setup

### 1. Install Flutter and Android tooling
1. Install the **Flutter SDK** (stable channel) and add it to your PATH: https://docs.flutter.dev/get-started/install/windows
2. Install **Android Studio** (easiest way to get the Android SDK, `adb`, and license acceptance): https://developer.android.com/studio
3. Run `flutter doctor` and resolve anything it flags (`flutter doctor --android-licenses` if prompted).

### 2. Generate the Android project scaffold
From the repo root:
```powershell
flutter create --platforms=android --org com.jaichand .
```
This adds the `android/` folder without touching the `lib/` code. It derives the app id `com.jaichand.break_reminder_app` from the `--org` plus `name: break_reminder_app` in pubspec.yaml — this must match the `kApplicationId` constant in `lib/core/db/database.dart` (it already does; don't change one without the other).

Afterwards, open `android/app/src/main/AndroidManifest.xml` and set:
```xml
<application android:label="Break Reminder" ...>
```

### 3. Get dependencies and run
```powershell
flutter pub get
flutter run
```
A real phone (USB debugging) is recommended — background/notification behavior only shows up reliably on a real device.

## Verification checklist
1. App launches to the home screen showing all-zero stats.
2. Settings → **Send test reminder now** → a heads-up notification appears with Start break / Skip actions.
3. Tap **Start break** → app opens into the stopwatch; wait a bit, tap **End break** → home shows breaks today = 1 and the time you took.
4. Send another test reminder, tap **Skip** → nothing appears in stats (skips are logged but not counted).
5. Kill the app mid-break, reopen → the stopwatch resumes from the original start time.
6. Set active days/hours to a window that excludes right now, end a break → no reminder should fire until the window opens again.
7. Toggle **Pause** → no reminders; resume → they're back.
8. Over the following days: reminders fire only on the selected days inside the selected hours, and survive a reboot.

## The data
Breaks are logged to a local SQLite database in the app's private storage — nothing leaves the device. Uninstalling the app deletes it.
