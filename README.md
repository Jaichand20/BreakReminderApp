# Squat Reminder (Android)

A lightweight Android port of [SquatReminder](https://github.com/Jaichand20/SquatReminder) — the Windows tray app that reminds you to do squats every hour. Every reminder fires as a notification with **Done ✓ (+10)** / **Skip** actions; a Done tap logs to a local SQLite database with no need to open the app. The app itself is a single-screen control panel: stat tiles, a week/month/year trend chart, and a GitHub-style year heatmap with streak/best-day/year-total callouts.

## What's already built
All of the Dart app code is written and ready in `lib/`:
- `lib/core/db/` — SQLite layer (`sqlite3` via FFI), ported 1:1 from the original's `squat_db.py` queries.
- `lib/core/notifications/` — hourly reminder scheduling + Done/Skip handling, including while the app isn't running.
- `lib/core/settings/` — pause/resume, interval, and squats-per-reminder, persisted via `shared_preferences`.
- `lib/features/home/` — the control panel screen (stat tiles, trend chart, heatmap).
- `lib/features/settings/` — pause/resume toggle, interval/squats steppers, "send test reminder now".

The Android-native project scaffold (`android/`) hasn't been generated yet — that's the first setup step below, since it needs to be generated by the Flutter tool itself to match whatever Flutter version you install (hand-writing Gradle files risks drifting out of date).

## Setup

### 1. Install Flutter and Android tooling
1. Install the **Flutter SDK** (stable channel) and add it to your PATH: https://docs.flutter.dev/get-started/install/windows
2. Install **Android Studio** — this is the easiest way to get the Android SDK, `adb`, and license acceptance done (needed even if you only use the command line afterward): https://developer.android.com/studio
3. Run `flutter doctor` in a terminal and resolve anything it flags (accept Android licenses with `flutter doctor --android-licenses` if prompted).

### 2. Generate the Android project scaffold
From this folder (`C:\Users\jaich\Desktop\MobileVersion`):
```powershell
flutter create --platforms=android --org com.jaichand .
```
This adds the `android/` folder (Gradle files, manifest, launcher icons, `MainActivity`) without touching the `lib/` code already here. It reads the existing `pubspec.yaml` (`name: squat_reminder`) plus the `--org` you passed to derive the app id `com.jaichand.squat_reminder` — this must match the `kApplicationId` constant in `lib/core/db/database.dart` (it already does; don't change one without the other).

Afterwards, open `android/app/src/main/AndroidManifest.xml` and set:
```xml
<application android:label="Squat Reminder" ...>
```
(it defaults to the raw project name otherwise).

**Sanity check:** confirm `pubspec.yaml` still lists `flutter_local_notifications`, `sqlite3`, `sqlite3_flutter_libs`, and `shared_preferences` under `dependencies`, and that `lib/main.dart` still matches what's in this repo — `flutter create` on an existing project is designed not to clobber these, but if it did, let me know and I'll restore them.

### 3. Get dependencies and connect a device
```powershell
flutter pub get
```
Then either:
- **Real phone (recommended)** — enable Developer Options + USB debugging on your Android phone, plug it in via USB, accept the debugging prompt on the phone. Real background/notification behavior only shows up reliably on a real device.
- **Emulator** — create one via Android Studio's Device Manager.

Check `flutter devices` shows your target, then:
```powershell
flutter run
```

## Verification checklist
1. App launches to the home screen showing all-zero stats.
2. Settings → **Send test reminder now** → a heads-up notification appears with Done/Skip actions.
3. Tap **Done ✓ (+10)** → notification dismisses without opening the app; reopen/pull-to-refresh the app and confirm Today's total went up by 10, and the trend chart / heatmap reflect it.
4. Send another test reminder, tap **Skip** → confirm nothing was logged.
5. Toggle **Pause** in Settings, send a test reminder → confirm none fires; **Resume** → confirm it does again.
6. Over the following hours/days on your own device: confirm reminders keep firing roughly hourly and survive a reboot.

## Changing the interval / squats-per-reminder
Open the app → Settings — both are adjustable there (default: every 60 minutes, 10 squats). No code changes needed.

## The data
Squats are logged to a local SQLite database in the app's private storage (not user-accessible without root/adb) — nothing leaves the device. Uninstalling the app deletes it.
